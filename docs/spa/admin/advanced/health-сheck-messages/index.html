OCR

Component name Type of issue Message Text
Microsoft SQL Server Error

MS SQL check failed: + e.getMessage()

Possible causes:

  1. OCR could not connect to Microsoft SQL Server.
  2. Microsoft SQL Server is down.
  3. The Microsoft SQL Server node is not reachable from ocr-rest.
  4. OCR Service is not operational.

Remedy:

Make sure:

  1. Microsoft SQL Server is up.
  2. The network between Microsoft SQL Server node and ocr-rest node is operational.
S3 Error

Failed loading a file: + e.getMessage()

Possible causes:

  1. S3 is not reachable from ocr-rest.
  2. OCR Service is not operative.

Remedy:

Make sure S3 is up.
S3 Error

Save operation failed during a storage test: + e.getMessage()

Possible causes:

  1. S3/MongoDB (depending on the active profile in configuration) is not reachable from ocr-rest.
  2. The connection is not configured correctly.
  3. MongoDB is down.
  4. OCR Service is not operational.

Remedy:

Make sure:

  1. S3 is up.
  2. The connection is configured properly.
RabbitMQ Error

Receive operation failed with an error during MQ test: ", e

Possible causes:

  1. RabbitMQ or MongoDB (depending on the active profile in configuration) is not reachable from ocr-rest or ocr-worker.
  2. RabbitMQ or MongoDB is down.
  3. ocr-worker is down.
  4. OCR Service is not operational.

Remedy:

Make sure:

  1. RabbitMQ or MongoDB is up.
  2. The network between RabbitMQ/MongoDB node and ocr-rest node is operational.
RabbitMQ Error

A bad message payload was received, expected='" + expectedPayload + "' actual='" + actual + "'"

Possible causes:

  1. An incompatible version of ocr-worker is deployed.
  2. Message Queue for Health Check has incorrect custom configurations.
  3. OCR Service is not operational.

Remedy:

Make sure:

  1. Compatible version if ocr-worker is deployed.
  2. ocr-worker is up and has the correct connection settings to RabbitMQ.
  3. Message Queue for Health Check is configured properly and consistent between ocr-rest and ocr-worker.
RabbitMQ Error

A message was not received during MQ testing, expected='" + expectedPayload + "', workers supposedly are not handling messages in MQ".

Possible causes:

  1. RabbitMQ/MongoDB (depending on the active profile in configuration) is not reachable from OCR-worker
  2. OCR-worker is not operational.
  3. Wrong version of ocr-worker is deployed.
  4. Message Queue for Health Check has incorrect custom configurations.
  5. OCR Service is not operational.

Remedy:

Make sure:

  1. RabbitMQ or MongoDB is up.
  2. ocr-worker is up and has the correct connection settings to RabbitMQ/MongoDB.
  3. Message Queue for Health Check is configured properly and consistent between ocr-rest and ocr-worker.
RabbitMQ Error Failed sending a message to RabbitMQ. Please, check that RabbitMQ is up and running. Refer to Installation Guide.
ocr-worker Error No reports from OCR-workers are found or the reports have expired. Please check number of the OCR-workers operating on the server and of the OCR-workers number is less then required start required OCR-worker number. Refer to WorkFusion OCR Guide.
