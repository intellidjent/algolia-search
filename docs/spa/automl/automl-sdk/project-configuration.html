Project Configuration

To configure an AutoML project, use either of the following two methods:

  1. Using an AutoML SDK Wizard (default)
  2. Using WorkFusion Studio
  3. Using the command line

Using an AutoML SDK Wizard

AutoML SDK Wizard is the simplest way to configure a new AutoML SDK project in WorkFusion Studio. Follow these instructions to quickly get started with a new project:

  1. In WorkFusion Studio, click File > New > AutoML SDK Project.
  2. Select the Archetype Catalog:
    1. Remote. Provide a URL to a pre-installed Nexus repository with all artifacts on your on-premise installation or a WorkFusion Maven repository for training purposes. Provide your username and password.
    2. Local. Provide a Path to the archetype-catalog.xml file.
  3. Select the way you want your project to be created:
    1. Create an empty project if you want to have an empty model without any configuration.
    2. Extend generic models if you want your model to have a generic model configuration.
  4. Select the Type of ML Project and AutoML SDK Version.
  5. Specify the properties and parameters for your AutoML SDK project. Note, that the parameters are already predefined.
    1. groupId — Group identifier of your artifact.
    2. artifactId — Unique identifier of your artifact.
    3. version — Artifact version.
    4. package — Default package for code generation.
    5. modelClassName — Class name of your model.
    6. modelCode — Hypermodel code, only [a-z, 0-9, _, -] are allowed for model code.
    7. modelDescription — Hypermodel description which will be used in WorkFusion during automation setup as a hint.
    8. modelTitle — Hypermodel title which will be used in WorkFusion during automation setup as value in drop-down.
    9. modelVersion — Hypermodel version.
  6. Click Finish and wait till your project is set up.

Now you're ready to start using your AutoML SDK project.

Using WorkFusion Studio

Project Setup

  1. In WorkFusion Studio, click Window > Preferences > Maven > Archetypes.
  2. Click Add Remote Catalog... to use a pre-installed Nexus repository with all artifacts on your on-premise installation or WorkFusion Maven repository for training purposes.
  3. Provide a URL to the repository, add repository description (optional), and then click OK.
  4. Click OK to save the changes.

Generating a Project from Archetype

  1. In WorkFusion Studio, click File > New > Other > Maven.
  2. Select Maven Project, and then click Next.
  3. Select Use default Workspace location, and then click Next.
  4. Select the Catalog that you created in the previous section, select the archetype for your use case (Information Extraction or Classification), and then click Next.
  5. Specify the Archetype parameters.
    1. groupId — Group identifier of your artifact.
    2. artifactId — Unique identifier of your artifact.
    3. version — Artifact version.
    4. package — Default package for code generation.
    5. importConfigurationFromGenericModel — enter y if you want your model to have generic model configuration or enter n if you want to have empty model without any configuration.
    6. modelClassName — Class name of your model.
    7. modelCode — Hypermodel code, only [a-z, 0-9, _, -] are allowed for model code.
    8. modelDescription — Hypermodel description which will be used in WorkFusion during automation setup as a hint.
    9. modelTitle — Hypermodel title which will be used in WorkFusion during automation setup as value in drop-down.
    10. modelVersion — Hypermodel version.
  6. Click Finish to generate a new project with the provided values.
  7. In the Project tree, right-click the pom.xml file.
  8. Click Run As > Maven install and wait for project to build.

As soon as you see the BUILD SUCCESS message in the Console, you are ready to use your custom model.

Using Command Line

To generate a stub using a command line, use one of the following Maven commands depending on the model type and Search Engine version.

Search Engine 1.0 Models

  1. Information Extraction
mvn archetype:generate -DarchetypeGroupId=com.workfusion.ml -DarchetypeArtifactId=ml-sdk-ie-archetype -DarchetypeVersion=<%artifact-version%>
  1. Classification
mvn archetype:generate -DarchetypeGroupId=com.workfusion.ml -DarchetypeArtifactId=ml-sdk-classification-archetype -DarchetypeVersion=<%artifact-version%>
  1. Binary Classification
mvn archetype:generate -DarchetypeGroupId=com.workfusion.ml -DarchetypeArtifactId=ml-sdk-binary-classification-archetype -DarchetypeVersion=<%artifact-version%>

Search Engine 2.0 Models

  1. Information Extraction
mvn archetype:generate -DarchetypeGroupId=com.workfusion.ml -DarchetypeArtifactId=ml-sdk-ie-se-20-archetype -DarchetypeVersion=<%artifact-version%>
  1. Classification
mvn archetype:generate -DarchetypeGroupId=com.workfusion.ml -DarchetypeArtifactId=ml-sdk-classification-se-20-archetype -DarchetypeVersion=<%artifact-version%>
  1. Binary Classification
mvn archetype:generate -DarchetypeGroupId=com.workfusion.ml -DarchetypeArtifactId=ml-sdk-binary-classification-se-20-archetype -DarchetypeVersion=<%artifact-version%>

Generating a Project

  1. Copy one of the above-mentioned commands.
  2. Replace <%artifact-version%> by the current version of your AutoML SDK (for example, 10.0.x.x).
  3. Next, provide the following fields:
    1. groupId — Group identifier of your artifact.
    2. artifactId — Unique identifier of your artifact.
    3. version — Artifact version.
    4. package — Default package for code generation.
    5. importConfigurationFromGenericModel — enter y if you want your model to have a generic model configuration or enter n if you want to have an empty model without any configuration.
    6. modelClassName — Class name of your model.
    7. modelCode — Hypermodel code, only [a-z, 0-9, _, -] are allowed for model code.
    8. modelDescription — Hypermodel description which will be used in WorkFusion during automation setup as a hint.
    9. modelTitle — Hypermodel title which will be used in WorkFusion during automation setup as value in drop-down.
    10. modelVersion — Hypermodel version.

Your Hypermodel stub is generated. It's a Maven project that can be imported in your favorite Java IDE.

The generated stub has following structure:

    └── pom.xml

Project Artifacts

In the generated stub, you can find the following classes:

  1. <%artifact-name%>Model — Class with a custom Hypermodel, here you can find a basic configuration of your Hypermodel — code, version, title, etc. Also here you can find generated annotation ModelConfiguration with pre-generated model configuration.
    The generated Hypermodel extends one of generic models - GenericIeHyperModel - for IE models and GenericMultiClassificationHyperModel - for Classification models.
  2. <%artifact-name%>Configuration — Class with your configuration for your model. It has methods like processor, featureExtractors, annotators where you can define your own model configuration.
  3. PostProcessorExample — Example of post-processor.
  4. FeatureExtractorExample — Example of feature extractor.
  5. ModelTrainingRunner — Class for local model training run.
  6. ModelProcessingRunner — Class for local model execution run, to develop/debug post-processing.
